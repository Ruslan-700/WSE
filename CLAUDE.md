## Project Overview

Warband Script Enhancer (WSE) — a C++/Lua modding framework that hooks into the Mount & Blade: Warband game engine to provide extended scripting operations, profiling, networking, and plugin support. It injects a DLL into the running game process and patches engine functions at runtime.

## Build System

Visual Studio 2013 solution (`WarbandScriptEnhancer.sln`), Win32 (x86) only, toolset v120_xp.

**Build from command line (Developer Command Prompt for VS 2013):**
```
msbuild WarbandScriptEnhancer.sln /p:Configuration=Release /p:Platform=Win32
```

**Build configurations** target different Warband editions via compile-time `#define` switches:
- `Release` / `Debug` — Vanilla Warband 1.174
- `ReleaseSteam` / `DebugSteam` — Steam edition (different memory addresses)
- `ReleaseDedicated` / `DebugDedicated` — Dedicated server (server-only ops)
- `ReleaseNoSSE` / `ReleaseSteamNoSSE` — No SSE for older CPUs

**Project dependency order:** `luajit` → `WSELib` → `WSELoader`. `WSEProfiler` is standalone C#/.NET (WinForms). `luaLanes` and `WarbandLib` are independent.

**Output:** Binaries go to `WSEBuild/` (e.g. `WSELib.dll`, `WSELibDedicated.dll`, `lua51.dll`).

## Architecture

### Context System (core pattern)

All functionality is organized into **context classes** inheriting from `WSEContext` (`WSELib/WSEContext.h`). Contexts receive lifecycle events (`Load`/`Unload`) and game events via `OnEvent(sender, WSEEvent, data)`.

Key event types: `ForceUnload`, `LoadOperations`, `ModuleLoad`, `GameLoad`, `OnFrame`, `OnSave`, `OnLoadSave`, `OnChatMessageReceived`.

### Operation Contexts

`WSEOperationContext` (subclass of `WSEContext`) is the base for all script operation providers. Each domain has its own context class (e.g. `WSEAgentOperationsContext`, `WSEItemOperationsContext`, `WSENetworkOperationsContext`). Operations use a prepare/extract/return pipeline:
- `Prepare()` — sets up operand descriptor
- `ExtractValue()` / `ExtractString()` / `ExtractBoundedValue()` — type-safe operand extraction
- `SetReturnValue()` — return result to script

### Hook System

`WSEHookManager` manages three hook types for patching the game engine at runtime:
- `WSEFunctionHook` — JMP redirect to callback
- `WSEJumptableHook` — replace jump table entries
- `WSEMemoryHook` — direct memory patches

Hooks are address-based and tied to specific game executable versions (Vanilla vs Steam vs Dedicated have different addresses).

### Lua Integration

LuaJIT is embedded as the scripting VM. The bridge layer lives in `WSELib/WSELua_*.cpp|h`:
- `WSELua_Callbacks.cpp` — Lua-to-game bridge callbacks
- `WSELua_Helpers.cpp` — data type conversion
- `WSELua_Iterators.h` — iterator implementations
- `WSELib/LuaGlobals.lua` — Lua standard library extensions

### Plugin System

Third-party DLLs can register via `WSEPluginInfo` struct providing `load`/`unload`/`event` callbacks.

## Key Directories

- **WSELib/** — Core framework: contexts, hooks, operations, Lua bridge, utilities (tinyxml2, muParser, json, SQLite3)
- **WarbandLib/** — Reverse-engineered game engine headers (~200 files). Structs use `assert_size<T>()` to verify layout matches the game binary
- **WSELoader/** — DLL injector that finds the Warband process and injects WSELib. Uses CRC32 to verify game version (1.174)
- **WSEProfiler/** — C# WinForms GUI for analyzing `.prof` files generated by `WSEProfilingContext`
- **WSESDK/** — Python module system addon files for modders (`header_operations_addon.py`, etc.)
- **luajit/** — Embedded LuaJIT build
- **luaLanes/** — Lua threading library
- **WSEExternal/** — Havok Physics SDK headers (read-only dependency)

## Important Conventions

- Version is defined in `WSELib/WSE.h` (`WSE_VERSION_MAJOR/MINOR/BUILD`, `WSE_NETCODE_VERSION`)
- Configuration is in `wse_settings.ini` at the repo root (logging, profiling, RCON, OGP, limits, Lua settings)
- WarbandLib structs must match the game's exact memory layout — always verify with `assert_size<>`
- Different game editions require different hook addresses — guard with preprocessor defines
- VCS is Git (converted from Mercurial)
